## 📋 API Specification

### Request
```json
POST /api/loyalty/issue
Content-Type: application/json

{
  "customerId": "CUST-123",           // Optional (one of customerId or customerPhone required)
  "customerPhone": "+1234567890",     // Optional (one of customerId or customerPhone required)
  "amountCents": 1000,                // Required, must be positive
  "source": "POS",                    // Optional: 'POS' | 'MANUAL' | 'REFUND' | 'ADJUSTMENT' (default: 'POS')
  "sessionId": "session-123",         // Optional
  "posTicketId": "ticket-456",        // Optional
  "stripeChargeId": "ch_123",         // Optional
  "metadata": {}                      // Optional JSON object
}
```

### Response (Success)
```json
{
  "success": true,
  "transactionId": "txn-789",
  "accountId": "acc-123",
  "balanceBeforeCents": 5000,
  "balanceAfterCents": 6000,
  "amountCents": 1000,
  "totalEarnedCents": 1500
}
```

### Response (Error)
```json
{
  "success": false,
  "error": "Error message"
}
```

**Status Codes:**
- `200` - Success
- `400` - Bad Request (invalid input, missing fields)
- `409` - Conflict (optimistic locking conflict)
- `500` - Internal Server Error

## 🔒 ACID Compliance

**Atomicity:**
- All database operations wrapped in `prisma.$transaction()`
- Either all operations succeed or all rollback

**Consistency:**
- Balance updates are validated (positive amounts)
- Account and wallet cache stay in sync

**Isolation:**
- Transaction isolation prevents concurrent conflicts
- Optimistic locking via `version` field (future enhancement)

**Durability:**
- Committed transactions are persisted
- Transaction log is immutable

## 🚀 Usage Example

```typescript
// Issue credits after POS reconciliation
const response = await fetch('/api/loyalty/issue', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    customerId: 'CUST-123',
    amountCents: 350, // 1% of $35.00 transaction
    source: 'POS',
    sessionId: 'session-123',
    posTicketId: 'ticket-456',
    stripeChargeId: 'ch_123',
    metadata: {
      transactionAmount: 3500,
      loyaltyRate: 0.01,
    },
  }),
});

const result = await response.json();
console.log(`Issued ${result.amountCents} cents. New balance: ${result.balanceAfterCents}`);
```

## 📊 Database Operations

**Transaction Flow:**
1. Find or create `LoyaltyAccount`
2. Create `LoyaltyTransaction` record
3. Update `LoyaltyAccount.balanceCents`
4. Update `LoyaltyAccount.totalEarnedCents`
5. Update `LoyaltyWallet` cache
6. Commit transaction

**Performance:**
- Single database transaction (atomic)
- Wallet cache for fast balance reads
- Indexed lookups by customer ID/phone
