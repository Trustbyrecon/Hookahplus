## 📋 API Specification

### Request
```json
POST /api/loyalty/redeem
Content-Type: application/json

{
  "customerId": "CUST-123",           // Optional (one of customerId or customerPhone required)
  "customerPhone": "+1234567890",     // Optional (one of customerId or customerPhone required)
  "amountCents": 500,                 // Required, must be positive
  "metadata": {                       // Optional JSON object
    "discountCode": "LOYALTY10",
    "sessionId": "session-789"
  }
}
```

### Response (Success)
```json
{
  "success": true,
  "transactionId": "txn-790",
  "accountId": "acc-123",
  "balanceBeforeCents": 6000,
  "balanceAfterCents": 5500,
  "amountCents": 500,
  "totalRedeemedCents": 500
}
```

### Response (Error)
```json
{
  "success": false,
  "error": "Error message"
}
```

**Status Codes:**
- `200` - Success
- `400` - Bad Request (invalid input, insufficient balance, missing fields)
- `403` - Forbidden (account not active)
- `404` - Not Found (account not found)
- `409` - Conflict (optimistic locking conflict)
- `500` - Internal Server Error

## 🔒 ACID Compliance

**Atomicity:**
- All database operations wrapped in `prisma.$transaction()`
- Either all operations succeed or all rollback

**Consistency:**
- Balance validation (sufficient balance check)
- Account and wallet cache stay in sync
- Cannot redeem more than available balance

**Isolation:**
- Transaction isolation prevents concurrent conflicts
- Optimistic locking via `version` field (future enhancement)

**Durability:**
- Committed transactions are persisted
- Transaction log is immutable

## 🚀 Usage Example

```typescript
// Redeem credits for discount
const response = await fetch('/api/loyalty/redeem', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    customerId: 'CUST-123',
    amountCents: 500, // $5.00 discount
    metadata: {
      discountCode: 'LOYALTY10',
      sessionId: 'session-789',
      discountType: 'percentage',
      discountValue: 10,
    },
  }),
});

const result = await response.json();
if (result.success) {
  console.log(`Redeemed ${result.amountCents} cents. New balance: ${result.balanceAfterCents}`);
} else {
  console.error(`Redemption failed: ${result.error}`);
}
```

## 📊 Database Operations

**Transaction Flow:**
1. Find `LoyaltyAccount` by customer ID or phone
2. Validate account exists and is active
3. Validate sufficient balance
4. Create `LoyaltyTransaction` record (type: REDEEM, amount: negative)
5. Update `LoyaltyAccount.balanceCents` (decrease)
6. Update `LoyaltyAccount.totalRedeemedCents` (increase)
7. Update `LoyaltyWallet` cache
8. Commit transaction

**Performance:**
- Single database transaction (atomic)
- Wallet cache for fast balance reads
- Indexed lookups by customer ID/phone

## 📝 Error Handling

**Validation Errors (400):**
- Missing `amountCents`
- Negative `amountCents`
- Missing both `customerId` and `customerPhone`
- Insufficient balance

**Not Found Errors (404):**
- Account not found by customer ID/phone

**Forbidden Errors (403):**
- Account exists but is not active

**Database Errors (500):**
- Transaction failures
- Constraint violations
- Connection errors

**Conflict Errors (409):**
- Optimistic locking conflicts (future enhancement)
- Concurrent transaction conflicts
