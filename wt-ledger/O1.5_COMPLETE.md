## 🔄 Integration Flow

### When Reconciliation Match Occurs:

1. **Reconciliation Match Found**
   - Stripe charge ↔ POS ticket matched
   - Confidence: 'high' (exact amount match within 1 minute)

2. **Customer Info Extraction**
   - Fetch session by `sessionId` from POS ticket or Stripe charge
   - Extract `customerRef` (customer ID) and `customerPhone`
   - Fallback: Extract from Stripe charge metadata/billing details

3. **Loyalty Credit Calculation**
   - Calculate: 1% of transaction amount
   - Minimum: 1 cent per transaction
   - Example: $35.00 transaction → 35 cents loyalty credit

4. **Credit Issuance**
   - Call `issueLoyaltyCredits()` helper function
   - Create `LoyaltyTransaction` with type 'ISSUE'
   - Update account balance atomically
   - Link to session, POS ticket, and Stripe charge

5. **Logging**
   - Success: Log credit issuance
   - Failure: Log warning (doesn't fail reconciliation)

## 📋 Implementation Details

### Customer Info Extraction Priority:

1. **Session Lookup** (Primary)
   ```typescript
   const session = await prisma.session.findUnique({
     where: { id: sessionId },
     select: { customerRef: true, customerPhone: true }
   });
   ```

2. **Stripe Charge Metadata** (Fallback)
   ```typescript
   customerId = charge.metadata?.customerId;
   customerPhone = charge.metadata?.customerPhone || charge.billing_details?.phone;
   ```

### Loyalty Calculation:

```typescript
// 1% of transaction amount, minimum 1 cent
const loyaltyRate = 0.01;
const loyaltyAmount = Math.round(transactionAmountCents * loyaltyRate);
return Math.max(1, loyaltyAmount); // Minimum 1 cent
```

**Examples:**
- $35.00 transaction → 35 cents loyalty credit
- $10.00 transaction → 10 cents loyalty credit
- $0.50 transaction → 1 cent loyalty credit (minimum)
