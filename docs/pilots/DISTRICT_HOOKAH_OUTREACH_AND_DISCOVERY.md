## First-call discovery checklist (30–45 minutes)
Capture answers inline; these map directly into LaunchPad + onboarding.

### A) Pilot scope and anchor selection
- **Confirm 3 pilot anchors** (recommended: Diablo’s Cantina @ MGM, The Bullpen DC, Opera Ultra Lounge).\n- **Go-live date** target and pilot start window.\n- **Success criteria** (pick 3–5 metrics; see `DISTRICT_HOOKAH_PILOT_METRICS.md`).\n- **Decision cadence**: weekly ops + week-4 exec readout.\n- **Rollout trigger**: what “yes” means for all-13 expansion.

### B) Operational reality (how hookah service actually runs)
- **Service model**:\n  - table-side sessions vs sections vs VIP “host-led” sessions\n  - coal management workflow (who, where, frequency)\n  - refill / flavor change workflow\n- **Session pricing**:\n  - flat vs timed\n  - standard vs premium tiering\n  - event-night surge pricing (if any)\n- **Session friction** today:\n  - where do delays happen?\n  - what causes mis-charges or disputes?\n  - what’s the biggest “venue partner complaint” risk?

### C) Data + integrations (POS vendor is currently unknown)
- **POS vendor per venue**: Square / Toast / Clover / none / other.\n- **Payment processor**: Stripe / Square / other.\n- **Today’s reconciliation**:\n  - how do you match hookah sessions to venue tickets?\n  - do you use receipt notes / external tender references?\n  - where do disputes happen?\n- **Data access**:\n  - can you export daily orders/payments from POS?\n  - do you have manager access to create API credentials if needed?\n\nIf POS vendor is unknown on the first call, schedule a 15-minute follow-up dedicated to POS discovery.

### D) Locations and capacity baselining (minimum required)
For each pilot location:\n- estimated tables or “active session capacity”\n- hours (especially peak windows)\n- staff roles (FOH, BOH, runner) and headcount per peak\n- typical session duration ranges\n- top 5 flavors + premium upsell candidates

### E) Stakeholders and access
- primary operator sponsor (sign-off)\n- ops lead (day-to-day)\n- training owner\n- venue partner contact(s) (if permitted)\n- who will hold the tablets / devices and manage credentials?

## Required artifacts (collect before LaunchPad Day)
- Pilot location list with table counts (or estimated capacity by section)\n- Flavor menu + premium tiers\n- Staff roster emails (or at least roles/count)\n- Photos or a sketch of floor/sections for each pilot venue\n- Current close-out/reconciliation checklist used by staff
